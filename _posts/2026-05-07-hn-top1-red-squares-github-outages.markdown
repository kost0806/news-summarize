### 개요

GitHub의 기여 그래프(초록 잔디)를 패러디한 프로젝트 **Red Squares**가 Hacker News 상단을 차지했다. 개발자 Cian Mullins가 만든 이 사이트는 초록색 커밋 표시 대신 빨간색 칸으로 GitHub.com의 공식 다운타임을 추적한다. GitHub의 잦은 장애 이력을 유머러스하게 시각화한 것으로, 커뮤니티에서 큰 호응을 얻었다.

### 주요 내용

- **역전의 기여 그래프**: 일반 GitHub 프로필에서는 코드 커밋이 초록 칸으로 채워지지만, Red Squares에서는 GitHub 공식 상태 페이지 기록을 기반으로 서비스 장애 시간이 빨간 칸으로 표시된다.
- **GitHub 장애 역사 시각화**: 지난 수년간 GitHub가 얼마나 자주, 얼마나 길게 다운됐는지를 한눈에 볼 수 있다. 최근 2026년 초 GitHub의 대규모 장애 이슈(CVE-2026-3854 등) 이후 더욱 주목받았다.
- **HN 반응**: 717포인트, 162댓글을 기록하며 상위에 올랐다. "GitHub가 이 그래프를 프로필에 보여줘야 한다"는 농담 섞인 댓글이 많았으며, GitHub 신뢰성에 대한 진지한 논의도 이어졌다.
- **기술적 구현**: GitHub 공식 상태 API(githubstatus.com)를 파싱해 실시간으로 업데이트된다.

### 시사점

Red Squares는 단순한 유머 프로젝트지만, 개발자 인프라로서 GitHub에 대한 의존도가 높아질수록 서비스 안정성이 얼마나 중요한지를 환기시킨다. 2026년 들어 GitHub의 가용성 문제가 수차례 화제가 됐던 만큼, 이 프로젝트는 플랫폼 신뢰성에 대한 업계의 집단적 불안감을 보여주는 거울이기도 하다.
