### 개요

"Computer, Enhance!" Substack을 운영하는 게임 프로그래머이자 성능 전문가 Casey Muratori가 1994년 마이크로소프트 하계 인턴십 면접에서 받은 프로그래밍 질문 네 개를 회고하는 글입니다. 2023년에 작성된 이 글이 최근 다시 주목받아 HN 상위권에 올랐습니다. 흥미롭게도 네 문제 중 최소 두 문제가 성능(performance) 관련 문제였다고 합니다.

### 주요 내용

#### 1994년 MS 면접의 특징

Muratori는 당시 마이크로소프트가 단순히 "코딩할 수 있는가"를 넘어 "코드가 얼마나 빠르게 실행되는가"를 묻는 면접을 진행했다고 회상합니다. 이는 현재 대부분의 코딩 면접이 알고리즘의 정확성과 공간/시간 복잡도를 묻는 것과는 다른 접근입니다.

#### 성능 중심 프로그래밍의 관점

Muratori는 "Performance-Aware Programming(성능 인식 프로그래밍)"이라는 커리큘럼을 운영하는 사람으로, 하드웨어가 실제로 어떻게 동작하는지 이해하는 것이 좋은 소프트웨어의 핵심이라고 주장합니다. 30년 전 MS 면접 문제들이 이 철학과 맞닿아 있다는 점이 인상적입니다.

#### 레트로 컴퓨팅과 현재

1994년은 Windows 3.1에서 Windows 95로 넘어가던 시기였습니다. 당시의 하드웨어 제약과 최적화 필요성이 프로그래머들에게 성능에 대한 깊은 이해를 요구했습니다. AI가 코드를 생성해주는 2026년, 우리는 오히려 하드웨어와 성능에 대한 이해를 잃어가고 있는 것은 아닐까요?

### HN 커뮤니티 반응

49점, 13개의 댓글로 레트로 컴퓨팅 팬들과 성능 프로그래밍 관심자들이 반응했습니다. "현대 면접이 얼마나 실제 프로그래밍 능력과 괴리됐는지", "30년 전 MS가 성능을 이렇게 중시했다는 것이 놀랍다"는 반응이 있었습니다.

### 시사점

30년 전의 면접 문제가 오늘날 HN에서 주목받는다는 사실은 성능 인식 프로그래밍의 영원한 가치를 보여줍니다. AI가 코드를 자동 생성하는 시대에, 하드웨어를 이해하고 성능을 의식하는 프로그래머의 희소성은 오히려 높아질 수 있습니다.
