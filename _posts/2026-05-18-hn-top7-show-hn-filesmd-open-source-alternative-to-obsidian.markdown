### 개요

개발자 zakirullin이 5년에 걸쳐 Go 언어로 개발한 로컬 우선(local-first), 프라이버시 중심의 오픈소스 노트 앱 Files.md를 공개했다. 모든 데이터를 순수한 마크다운(.md) 파일로 저장하며, 설치 없이 브라우저에서 실행되고 오프라인으로 동작한다. Obsidian의 오픈소스 대안으로 주목받으며 GitHub 스타 925개를 기록했다.

### 주요 내용

#### Files.md의 핵심 철학

Files.md의 설계 철학은 "소유권과 단순함"이다. 데이터는 항상 사용자가 소유하는 순수한 텍스트 파일로 존재하며, 어떤 독점 포맷이나 서비스에도 종속되지 않는다.

#### 주요 기능

**데이터 저장 방식**
- 모든 노트는 표준 마크다운(.md) 파일로 저장
- 어떤 텍스트 편집기로도 열고 수정 가능
- 벤더 종속(vendor lock-in) 완전 배제

**실행 환경**
- **설치 불필요**: 브라우저에서 직접 실행
- **오프라인 동작**: 인터넷 연결 없이도 완전한 기능 사용 가능
- Go로 작성된 단일 바이너리(single binary)로 배포

**동기화 및 확장**
- 선택적 자체 호스팅 동기화 서버: **단일 Go 바이너리**로 제공
- **Telegram 챗봇 연동**: 모바일에서 빠른 노트 저장 가능
- LLM(Large Language Model)으로 확장 가능하도록 설계

**설계 원칙**
- AI 없음 (No AI built-in)
- 트래킹 없음 (No tracking)
- 분석 수집 없음 (No analytics)

#### Obsidian과의 비교

| 항목 | Files.md | Obsidian |
|------|----------|---------|
| 라이선스 | 오픈소스 | 비상업적 무료/유료 |
| 데이터 포맷 | 순수 마크다운 | 마크다운 (플러그인 의존) |
| 동기화 | 자체 호스팅 가능 | Obsidian Sync 유료 |
| 설치 | 불필요 (브라우저) | 데스크톱 앱 필요 |
| AI 기능 | 없음 (외부 LLM으로 확장) | 플러그인 생태계 |
| 트래킹 | 없음 | 있음 |

#### 기술 스택

- **언어**: Go
- **배포**: 단일 바이너리
- **프론트엔드**: 브라우저 기반
- **GitHub 스타**: 925개

**Obsidian 사용자들의 관심**: 많은 Obsidian 사용자들이 동기화 비용과 폐쇄적 생태계에 불만을 가지고 있었기에, 오픈소스 대안에 큰 관심을 보였다.

**"로컬 우선" 철학 지지**: 데이터를 순수한 파일로 소유한다는 철학이 많은 공감을 얻었다. 특히 서비스 종료나 구독료 인상에 대한 불안 없이 노트를 영구적으로 소유할 수 있다는 점이 높이 평가되었다.

**Telegram 연동의 참신함**: 모바일 캡처를 위한 Telegram 챗봇 연동 아이디어가 특히 주목받았다. 별도의 모바일 앱 없이도 빠른 노트 저장이 가능하다는 점이 창의적이라는 평가를 받았다.

**기능 요청 및 개선 제안**: 더 풍부한 편집기(WYSIWYG), 그래프 뷰, 백링크(backlink) 지원 등 Obsidian의 인기 기능들을 요청하는 댓글이 많았다. 일부는 현재의 단순함이 오히려 강점이라고 반박했다.

**5년 개발의 가치**: 개인 프로젝트를 5년간 꾸준히 발전시킨 것 자체에 대한 경의를 표하는 댓글도 여럿 있었다.

### 시사점

Files.md는 AI와 구독 경제가 지배하는 시대에 역행하는 "단순함과 소유권"이라는 가치를 내세우며 주목받고 있다. 개인 지식 관리(PKM, Personal Knowledge Management) 도구 시장에서 오픈소스와 로컬 우선 접근법이 강력한 대안이 될 수 있음을 보여준다. 5년간의 개인 프로젝트가 HN에서 500점 이상을 받으며 주목받은 것은, 화려한 기능보다 명확한 철학과 진정성이 개발자 커뮤니티에서 통한다는 점을 보여준다.
